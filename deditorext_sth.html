About

Small Tools Hub (STH) its a handful editor tool crated for integration of small tools those need to run in the editor but not deserve its own window to be stated.

The module supplies you with easy to use, still powerful API that allows you to embed a tool into Unity without digging into backstage processes.

Read also: Custom plug-in development

How does it work

The tool works as a EditorWindow that envelops Unity services and simplifies them access via own API.

The tool scans entire in-project solutions for STHPlugin derived classes and loads them instances as plug-ins.

Entire messages excluding the OnInspectorGUI calls for each enabled plug-in.
Inspector GUI calls only when the plug-in interface selected at the inspector.

How to..

Start

Small Tools Hub can be started via top editor menu panel by following path:
Doloro GDK \ Small Tools Hub

Warning
On some platforms or package versions the Doloro GDK category can be a sub-category of Tools category.


Enable \ Disable plug-in

Each plugin within Small Tools Hub has a toggle at the left side of its catalog item.

Toggle state defines is the plug-in receives editor messages.

The only callback that works not matter status is, is a OnInspectorGUI over your plug-in, that draws GUI within STH window when the plug-in has been selected in catalog.

That allows you to disable tools you no need at the moment.


Configure a plug-in

To enter the plug-in's GUI you just need to select it in STH's catalog.
After that the right side of the window will draw UI provided with a OnInspectorGUI supplied with the plug-in.
