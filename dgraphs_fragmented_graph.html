About

It's kind of possible situation when you need to navigate between several graphs like a whole.
By default the graphs not allows that and during pathfinding cross several graphs you will get an exception caused by inability to build the path.

For such cases Doloro Graphs module supplies you with a built above module that allows to build a path cross several supportive graphs.

To be compatible the graph must be derived from a FragmentedGraph class, added to the GraphNetwork instance and navigated through it.

Architecture

Fragment graph

Fragment graph is a extended Graph class that allows to defined graph nodes used and enter \ escape points during cross graphs navigation.

Such points define which node in graph represents which side in world space and allows to combine fragmented graphs to a network automatically.

Explore API: Doloro.Graphs.Fragmented.AFragmentGraph : Graph


Graphs network

Graphs network by itself is also the Graph. But instead of working with a regular GraphNode instances it uses other Graph instances as nodes.

To add any graph to the network, the graph must be derived from the Fragment graph.
During graphs connection you are selecting which sides of graphs connects. That allows to build a whole graph in intuitive square tiles like format.

Explore API: Doloro.Graphs.Fragmented.GraphsNetwork : Graph

How to...

Create network

To create a new network enough to call new GraphsNetwork() and sign FragmentGraph instances into it.

Following example demonstrates connection of 4 graphs into mutual network.

// Creating and filling graphs that will be a part o network.
FragmentGraph graph00 = ...;
FragmentGraph graph01 = ...;
FragmentGraph graph10 = ...;
FragmentGraph graph11 = ...;
// Creating a network.
var network = new GraphsNetwork();
// Connecting graphs to each other.
// 01 11
// 00 10
network.Connect(graph00, graph01, Direction.Top);
network.Connect(graph00, graph10, Direction.Right);
network.Connect(graph01, graph11, Direction.Right);
network.Connect(graph10, graph11, Direction.Top);

You also can connect nodes of graphs in network manually using following pattern. In case if graphs not included to the network yet, the system will make this appointment automatically.

var node0 = graph00[NODE_SIGNATURE_0];
var node1 = graph11[NODE_SIGNATURE_1];
network.Connect(node0, node1);

where NODE_SIGNATURE_0 and NODE_SIGNATURE_1 are signature of nodes that allows to access them in graph.


Build cross-network path

To navigate cross graphs you enough to request path build via GraphsNetwork API.

var path = network.BuildPath(
graph00[NODE_SIGNATURE_0], Direction.Top,
graph11[NODE_SIGNATURE_1], Direction.Bottom);

or

var path = await network.BuildPathAsync(
graph00[NODE_SIGNATURE_0], Direction.Top,
graph11[NODE_SIGNATURE_1], Direction.Bottom);
