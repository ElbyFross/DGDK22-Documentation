Creation

For plug-in create its enough to create a new class derived from the STHPlugin class.
See also: Doloro.EditorExtension.SmallToolsHub.STHPlugin

Warning
The class MUST be located at the editor folder.

The new plug-in class will be automatically found among loaded in-project assemblies and connected to the Small Tools Hub.

using Doloro.EditorExtension.SmallToolsHub;
public class NewPlugin : STHPlugin
{
// Override members using at your plug-in logic here.
}

Plug-in settings

STHPlugin allows you to define intergration settings that will be read by the manager during the operation.

Order in catalog

To setup an order of plug-in in catalog window override order property and define required value.
Higher value means later call.

Order also defines order in receiving of editor messages. If some of your plug-in based on other one, be sure it has a higher order than the base one.

See also: Doloro.EditorExtension.SmallToolsHub.STHPlugin.order


Header GUIContent

You may declare a custom GUIContent to display at plug-in header \ catalog item GUI elements.

To set your own content you have to override content property.
To drop the content and turn on auto naming, set the property to null.

See also: Doloro.EditorExtension.SmallToolsHub.STHPlugin.content


State setup via code

You may setup a plug-in activity state following the next pattern.

// Getting plug-in instance.
var plugin = STHPlugin.GetPlugin<PLUGIN_TYPE>();
// Requesting a state:-----------------------------------------
// To enable plug-in execution.
STHEditor.settings.MarkPluginAsEnalbed(plugin.SystemName);
// OR
// To disable plug-in execution.
STHEditor.settings.MarkPluginAsDisabled(plugin.SystemName);

Cross-session data

For most plug-ins its important to store some data between editor using sessions.
STH tool provides you to follow a simple pattern to do this.

  1. Create a new class at runtime assembly (not Editor folder).
  2. Derive it from ScriptableObject.
  3. Define members to store. Be sure that private and protected members has SerializeField attribute.
    public ExampleDataContainer : ScriptableObject
    {
    public string prop0
    {
    get => _prop0;
    set => _prop0 = value;
    }
    public int field0;
    [SerializeField] private _prop0;
    }
  4. Declare container class at the plug-in source scope.
  5. Implement DataSave and DataLoad methods as suggested below:
    using Doloro.EditorExtension.SmallToolsHub.Plugins;
    public class NewPlugin : STHPlugin
    {
    // Name of the container asset file to use.
    const string CONTAINER_ASSET_NAME = "ExampleDataContainerFileName";
    public CameraControlDataContainer container
    {
    get
    {
    if (_container == null) DataLoad();
    return _container;
    }
    set => _container = value;
    }
    private CameraControlDataContainer _container;
    public override void DataLoad() =>
    this.DataLoad(ref _container, CONTAINER_ASSET_NAME);
    public override void DataSave() =>
    this.DataSave(ref _container);
    }

Now you data container will be operated automatically by the tool via Doloro GDK cache.
The instance of data container will be always exists by container property call.

On-scene GUI

STH allows you simple access to the on scene scope view with a custom GUI.

Main toolbar

The tool automatically integrates its own extra toolbar over scene view that allows you to integrate frequently used controls on the scene.

All you need to do to place your GUI at the toolbar it's just override a STHPlugin.OnSceneGUIMainToolbar handler.

using Doloro.EditorExtension.SmallToolsHub;
public class NewPlugin : STHPlugin
{
public override void OnSceneGUIMainToolbar(STHEditor editor)
{
// Set your own editor GUI using EditorGUILayout features.
}
}

See also: Doloro.EditorExtension.SmallToolsHub.OnSceneGUIMainToolbar


Custom layout

You may access free scene view layout by overriding STHPlugin.OnSceneGUI hadnler.

using Doloro.EditorExtension.SmallToolsHub;
public class NewPlugin : STHPlugin
{
public override void OnSceneGUI(STHEditor editor)
{
// Set your own editor GUI using GUI and EditorGUI features.
}
}

See also: Doloro.EditorExtension.SmallToolsHub.OnSceneGUI
