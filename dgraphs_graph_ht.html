Create graph

To create a new empty graph you need to follow next pattern.
var graph = new Doloro.Graphs.Graph();

Create node

You may create a new graph node in several ways.

using Doloro.Graphs;
// Creating a graph.
var graph = new Graph();
// ------------------------------------------------------
// OPTION 1
// Assignation via node's constructor.
var node1 = new GraphNode(graph);
// ------------------------------------------------------
// OPTION 2
// Delayed assignation.
var node2 = new GraphNode();
// > Doing smth with node before assignation.
// >
// >
// Assigning node to the graph.
graph.Add(node2);
// ------------------------------------------------------
// OPTION 3
// Automatically creating node directly on the graph.
//
// New default `GraphNode`.
GraphNode node3 = graph.Add();
// New `GraphNode` with defined signature.
GraphNode node4 = graph.Add(CUSTOM_SIGNATURE);
// New node of custom node type derived from `GrpahNode`
CUSTOM_TYPE node5 = graph.Add<CUSTOM_TYPE>();

All the options leads to the save result but useful for different cases of nodes handling processes.

See also:

  • Doloro.Graphs.Graph
  • Doloro.Graphs.GraphNode

Custom node types

The graph node not limited with default implementation.
Fill free to derive a custom type from GraphNode class and implement extra logic and members if you need.

Get node

Each node has its own signature that allows to access it on graph.
See also: Doloro.Graphs.GraphNode.signature

The signature may be defined explicitly during node creation via constructor.
In case if has no been defined will be generated automatically.

Access to the GraphNode by its signature goes in element access format:

using Doloro.Graphs;
// Creating a graph with node signed as "node0"
string CUSTOM_SIGNATURE = "node0";
var graph = new Graph();
var node0 = new GraphNode(graph, CUSTOM_SIGNATURE);
// Getting the node by its key.
var receivedNode0 = graph[CUSTOM_SIGNATURE];
// node0 equals to receivedNode0
// Safer way to get node
if(graph.TryGetNode(CUSTOM_SIGNATURE, out var node))
{
// Not has been found.
// Use in case you aren't sure it exists.
}

See also: Doloro.Graphs.Graph.TryGetNode

Connect nodes

For connection of two nodes you enough to call them following next pattern.

// Creating graph.
var graph = new Graph();
// Adding nodes into it.
var node0 = graph.Add();
var node1 = graph.Add();
// Connecting code.
node0.Connect(node1);

During connection you may:

  • Define connection weight.
  • Define is the connection work in both way or only from origin node to destination.
  • Apply custom arguments to Connection (could be used in custom connection as initialization data).

See also: Doloro.Graphs.GraphNodeAPI.Connect

Disconnect nodes

  • You may disconnect one node from another following a simple pattern:
    node.Disconnect(otherNode);
  • You may also release entire connection of the node by call in format:
    node.DisconnectAll();
  • You may call disconnection for a certain Connection instance by call in format:
    connection.Disconnect();

See also:

  • Doloro.Graphs.GraphNodeAPI.Disconnect
  • Doloro.Graphs.GraphNodeAPI.DisconnectAll

Get connection

  • You may get Connection instances between two nodes by call in format:
    Connection[] cons = fromNode.GetConnections(toNode);
  • You may get entire Connection instances of the node by call in format:
    Connection[] cons = node.connections;
  • You may get certain Connection instance from Graph by its id if you know it, by call in format:
    Connection con = graph.GetConnection(connectionId);

See also

  • Doloro.Graphs.GraphNode.GetConnections
  • Doloro.Graphs.GraphNode.connections
  • Doloro.Graphs.Graph.GetConnection

Set weight connection weight

The very popular segment of graphs in discrete math is weighted graphs.
Doloro Graphs allows you to define weights for connections.

via constructor

You may define connection weight when you are creating the connection.

int CONNECTION_WEIGHT = ___ // < Set your value.
nodeFrom.Connect(nodeTo, CONNECTION_WEIGHT);

Instance

You may change the connection weight after it has been established by access to its weight property.

// Creating connection between nodes.
var connection = nodeFrom.Connect(nodeTo);
int CONNECTION_WEIGHT = ___ // < Set your value.
connection.weight = CONNECTION_WEIGHT;

See also: Doloro.Graphs.Connection.weight

Set duplex connection state

By default entire connections goes as duplex (works in two way).
But normal case in graphs math when the connection works only in one way. For this you have to turn off duplex mode on connection.

via constructor

You may define duplex mode state during connection the creation.

boo IS_DUPLEX_ON = ___ // < Set your value.
nodeFrom.Connect(nodeTo, IS_DUPLEX_ON);

Instance

You may change the duplex mode after connection has been established by access to its isDuplex property.

// Creating connection between nodes.
var connection = nodeFrom.Connect(nodeTo);
boo IS_DUPLEX_ON = ___ // < Set your value.
connection.isDuplex = IS_DUPLEX_ON;

See also: Doloro.Graphs.Connection.isDuplex

Define node's world position

As soon as we are talking about graphs using in games development, its a regular task to be able to use graph not only like a data set, but also as part of the in-game world (UI, Pathfinding, Maps, etc).

Doloro Graphs allows you to make a bond between a GraphNode and its in-world representation coordinates.

// Creating graph.
var graph = new Graph();
// Adding node into it.
var node = graph.Add();
// Defining world position of node's
// in-world representation.
Vector3 WORLD_POS = ___ //
// Making bond between node and the position.
node.SetPostion(WORLD_POS);

To get the stored node's position use the following pattern.

var storedWorldPos = node.GetPostion();
