About

GUI elements and related features, and systems related to Doloro Inventory module.

Controls

  • StorageGUIElement: GUI controller that makes a bond between StoragesDatabase data containers and front-end GUI elements.
  • ItemGUIElement: GUI controller that makes a bond between back-end database related to Item resource and front-end GUI element.
  • StoreGUIElement: UI controller that consumes two ExchageBufferGUIElement elements and handles extra store related controls to provide transaction related operations.
  • ExchageBufferGUIElement: Implemented in way of 2 UI blocks. First contains relevant storage content available for transfer. Second one contains items reserved with transaction orders between c current and destination storages.
  • StoragesSelectionGUIElement: GUI controller that provides selection between several Storages.

GUI extensions

The inventory entities can has defined custom properties that extends their features. To connect such a property should be used a custom component derived from AEntityPropertyGUIExtension<T> where T is a type of target entity (Item or Storage as example).

How To

Make cross window bond

Doloro UI Engine II implements conception of layouts that could be connected with virtual chain. That conception allows to create flexible dynamic UI but can't proof a strong reference to child components.

Basing on the fact you shouldn't develop direct UI command execution in case if you not sure that your implementation is final and can't has any followers.

Recomendations According with this we recommend to use next options:

  • Standalone GUI controller that manages events related to certain GUI feature.
    • Cons: Stable architecture.
    • Props: Not allows to create several GUI control made as standalone. The will conflict each other.
    • Example: Storages selection menu is allowed as single instance and has static API that informs subscribed controls about selection changes.
  • Global messenger that shares signed events to any subscribed component.
    • Cons: Flexible architecture.
    • Props: Each call causes necessary checks.
    • Example: When storage selected in list parent controller shares message with its signature and selected element. Every subscribed on messenger controller receives message and checks if it addressed to it.
  • Direct search from child control up to virtual hierarchy chain parents and subscribing on events
    • Cons: Any count of instances. Flexible architecture.
    • Props: High-weight search for each control for finding of parent. Hierarchy to parent controller must be strait and not allows parallel branches.
    • Example: Inspector looking for parental selection controller and the listen to selection events to updata its content.

Get an enity property

As soon as inventory system works via ECS a rented entity could take a different source at different time. So you should follow events to get relevant data.

Following pattern allows you to update your UI with relevant data as soon as something changed without unnecessary checks.

// Field that will hold handling property instance.
private PROPERTY_TYPE property;
// Connecting to inspector events.
protected virtual void Start()
{
inspector.ContentChanged += OnStorageChanged;
}
// Releases subscription as soon as object dead.
protected virtual void OnDestroy()
{
try { inspector.ContentChanged -= OnStorageChanged; }
catch { } // Object already destroyed and can't be accessed.
}
// Occurs as soon as instance that rents out entity is changed.
private void OnStorageChanged(AInspectorGUIElement<Storage> obj)
{
if(property != null)
{
try { property.Modified -= OnPropertyModified; }
catch { } // Property can't be accessed.
}
property = inspector.content.GetComponent<PROPERTY_TYPE>();
if(property != null)
{
// Subscribing on property modifying event to stay along with relevant values.
property.Modified += OnPropertyModified;
}
}
// Loading relevant data to UI as soon as source updated.
private void OnPropertyModified(EntityProperty __) => LoadGUIContent();

PROPERTY_TYPE is a type of property targeted by that GUI extension.
