About

Standardizes a flexible trigger system taht allows collecting and filtering objects passing through a collider zone.

Sensor types

<tt>Sensor</tt>

Primitive sensor can establish demanded physics envirounment. Detectes the objects coliinding with the child trigger zones.

<tt>FilteredSensor</tt>

Derived from the Sensor class this implementation allows to filter detecting objects passing through only the ones who goes along with filter rules.

<tt>DependentFilteredSensor</tt>

The sensor derinved from the FilteredSensor. Not hadnles physics itself, but following the events of another FilteredSensor derived instance to make extra filtered collection along with filters result of the source.

Use to prvent extra colliders using. This sensor allows to use the same trigger zone any count of times.

External filters

About

The system allows to bond custom extra filters to extend basic rules.

Bundled filters

  • Hierarchy Root Sensor Filter - filters objects by hierarchy check.
  • Layer Sensor Filter - filters objects along with the GameObject.layer property.
  • Tag Sensor Filter - filters objects along with the GameObject.tag property.
  • Game Object Sensor Filter - filters the ceratin game object isntance.

New filter creation

  • Create a MonoBechavior component and inheirt it from the ASensorFilter interface.
  • Add the component on the same object as the target SensorManager.
  • The new filter will be loaded during the sensor awaking or manual via ReloadFilters API.

Development rules

  • Call ASensorFilter.OnModified handler each time when you modifies core properties of your filter in case if you what to update your filtered colletion alnong with new rules.
  • You no need to implement ASensorFilter.Inverted property into your filter's logic. It will auto applyied into the ASensorFilter.CheckCollision handler calling by the sensors.
