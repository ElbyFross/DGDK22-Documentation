Database

A data related to the storages state holding by the single tone database accessible by the StoragesDatabase API. That data could be serialized to binary view and saved on the customer's computer for cross-session using.

ECS

Inventory system has implemented Entity-Component-System conception that recomemended to be used via the RuntimeStoragesHelper service. In that case any Storage within in-game world wold exist only when you need it. Any other time its content will be stored inside the StoragesDatabase without creating garbage on the scene and damaging of customer's computer performance.

Remarks The service always holds at least one entity of ones loaded resource on the scene even if it have been released from renting. That made to prevent uneccessaty often instantiations during each call. In case if you trying to rent entity but the single-tone entity already rented the system will create temporary entity that will be destroyed immediately as soon as released.

How to rent entity

There is several ways to rent an enity but the traditional one is to ask for enity with the Storage.id provided by the StoragesDatabase during storage registration.

Guid storageId = YOUR_ID;
var storageEntity = Doloro.InvetorySystem.RentStorrageEntity(storageId);

How to release enity

When you are done with entity access and it's not required anymore you hist need to release it with the same source as it was rented.

Guid storageId = YOUR_ID;
Doloro.InvetorySystem.ReleaseRentedEntity(storageId)
